# Nova — полный backlog возможностей

## Главный принцип

- Не переписывать существующее ядро.
- Новые внешние интеграции подключать через MCP.
- Системные Windows-функции оставлять обычными Tool Calling tools.
- Сначала искать готовый MCP-сервер.
- Любое действие должно иметь verification.
- Любой side effect должен быть защищён от повторного выполнения.
- Простые команды выполнять без LLM.
- Один пользовательский запрос — минимум модельных вызовов.
- Не подключать GitHub Models.
- Не считать наличие файла или класса доказательством готовой функции.
- Каждую новую возможность подтверждать тестом и ручным сценарием.
