## Тестирование MCP (Model Context Protocol)

### 1. MCP Gateway - базовая функциональность

**Что проверить:** MCP Gateway должен успешно инициализироваться и регистрировать инструменты.

**Ожидаемый вывод в консоли:**
```
INFO | MCP tools integrated. Available tools: X
```

Где X - количество доступных MCP инструментов (зависит от настроенных токенов в .env).

### 2. MCP Server Integration

Если в `.env` настроены токены:
- `GITHUB_TOKEN` - проверьте работу GitHub MCP
- `SLACK_TOKEN` - проверьте работу Slack MCP  
- `GOOGLE_DRIVE_TOKEN` - проверьте работу Google Drive MCP
- `JIRA_TOKEN` - проверьте работу Jira MCP
- `MCP_SQLITE_PATH` - проверьте SQLite MCP
- `MCP_POSTGRES_CONNECTION` - проверьте PostgreSQL MCP

**Фразы для проверки:**
- "Найди репозитории на GitHub" (GitHub MCP)
- "Покажи мне каналы в Slack" (Slack MCP)
- "Найди файлы в Google Drive" (Google Drive MCP)
- "Создай задачу в Jira" (Jira MCP)
- "Выполни SQL запрос SELECT * FROM users" (SQLite MCP)
- "Запусти контейнер nginx" (Docker MCP)
- "Покажи статус git репозитория" (Git MCP)

## Тестирование Reasoning Loop

### 1. Базовое reasoning

**Фраза для проверки:** "Реши задачу по шагам: если x + 5 = 12, то найди x"

**Ожидаемый результат:** Агент должен выполнить итеративный reasoning и прийти к ответу x = 7.

### 2. Параллельное выполнение

**Фраза для проверки:** "Сложи 2 + 2, умножь 3 * 3 и найди корень из 16"

**Ожидаемый результат:** Все три операции должны выполниться параллельно.

### 3. Self-reflection

**Фраза для проверки:** "Сколько будет 5 * 6?"

**Ожидаемый результат:** Агент должен вычислить 30 и проверить результат.

## Тестирование Recovery & Self-healing

### 1. Graceful degradation

**Фраза для проверки:** При ошибке базы данных
- Агент должен переключиться в degraded режим
- Некоторые инструменты должны стать недоступны

### 2. MCP fallback

**Фраза для проверки:** При ошибке модели
- Система должна попытаться использовать MCP инструменты как fallback

## Логирование

**Где смотреть:** Все операции логируются в консоль и файл.

**Ключевые логи:**
- `MCP Gateway initialized` - успешная инициализация MCP
- `MCP tools registered` - инструменты зарегистрированы
- `Reasoning loop started` - запущен цикл reasonin'а
- `Tool composition executed` - выполнена композиция инструментов

## Что делать при ошибках

1. **Скопируйте лог ошибки** - вставьте в отчет
2. **Укажите фразу** - что именно вы сказали
3. **Опишите ожидание** - что должно было произойти
4. **Укажите версию** - какой коммит использовался
