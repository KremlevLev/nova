
### Шаг 1. Архитектура папок (Скелет)

Создайте папку для проекта (например, `JARVIS-OS`) и выстройте внутри следующую структуру. Каждая папка отвечает за одну конкретную задачу (принцип единственной ответственности).

```text
JARVIS-OS/
│
├── core/                   # Ядро и конфигурация
│   ├── __init__.py
│   ├── config.py           # Глобальные настройки (пути, имена моделей, API-ключи если нужны)
│   └── prompts.py          # Системные промпты для LLM
│
├── modules/                # Основные рабочие модули
│   ├── __init__.py
│   ├── audio/              # "Уши и Голос"
│   │   ├── __init__.py
│   │   ├── stt.py          # Speech-to-Text (Vosk / Whisper)
│   │   └── tts.py          # Text-to-Speech (pyttsx3 / Coqui)
│   │
│   ├── brain/              # "Мозг"
│   │   ├── __init__.py
│   │   ├── llm.py          # Клиент общения с Ollama
│   │   ├── memory.py       # Работа с RAG и ChromaDB
│   │   └── router.py       # semantic_router для классификации команд
│   │
│   ├── tools/              # "Руки" (Инструменты для Function Calling)
│   │   ├── __init__.py
│   │   ├── os_utils.py     # Управление окнами, процессами (psutil, pywin32)
│   │   ├── tasks.py        # Взаимодействие с schtasks (планировщик)
│   │   └── registry.py     # Регистратор доступных функций для LLM
│   │
│   └── ui/                 # "Лицо"
│       ├── __init__.py
│       └── overlay.py      # Виджет на CustomTkinter или Flet
│
├── data/                   # Папка для локальных данных (в .gitignore)
│   ├── chroma_db/          # Векторная база данных
│   └── temp/               # Временные файлы (скриншоты, аудио-буфер)
│
├── main.py                 # Главная точка входа (оркестратор)
├── requirements.txt        # Список зависимостей
├── .gitignore              # Игнорирование venv, кэша и папки data
└── README.md               # Сюда пойдет ваш манифест
```

### Шаг 2. Как мы будем это разрабатывать (План-стратегия)

Чтобы не выгореть и видеть результат сразу, предлагаю двигаться итеративно. Вот идеальный порядок разработки (по этапам):

1. **Фундамент:** Создаем структуру выше, настраиваем виртуальное окружение.
2. **Мозг (Текстовый режим):** Пишем `brain/llm.py`. Подключаемся к локальной Ollama, учимся отправлять текст и получать ответ. Пока без голоса, тестируем в консоли.
3. **Руки (Function Calling):** Пишем 2-3 простые функции в `tools/os_utils.py` (например, узнать текущее время, открыть Блокнот). Учим Ollama вызывать их через JSON/Tools API.
4. **Маршрутизация:** Внедряем `semantic_router`, чтобы отличать обычный разговор ("как дела?") от системных команд ("закрой браузер").
5. **Голос и Слух:** Пишем модули STT и TTS. Соединяем это с текстовым мозгом. Джарвис начинает слышать и говорить.
6. **Долгая Память:** Подключаем `ChromaDB` для сохранения контекста.
7. **Пользовательский интерфейс (UI):** Оборачиваем всё в красивый плавающий виджет `CustomTkinter`.

### Ваши действия прямо сейчас (Практика)

Давайте заложим фундамент. Откройте терминал (Командную строку или PowerShell) и выполните следующие команды:

**1. Создаем папку и виртуальное окружение:**
```bash
mkdir JARVIS-OS
cd JARVIS-OS
python -m venv venv
```

**2. Активируем окружение:**
*Для Windows PowerShell:*
```powershell
.\venv\Scripts\Activate.ps1
```
*Для обычной командной строки (cmd):*
```cmd
venv\Scripts\activate.bat
```

**3. Создаем файл `requirements.txt`:**
Создайте этот файл в корне проекта и вставьте туда базовые библиотеки для начала:

```text
# LLM и Маршрутизация
ollama
semantic-router
langchain

# Инструменты Windows
psutil
pywin32
pyautogui

# Голос (пока базовый)
SpeechRecognition
pyttsx3
vosk
sounddevice

# Память
chromadb
```

**4. Устанавливаем библиотеки:**
```bash
pip install -r requirements.txt
```
